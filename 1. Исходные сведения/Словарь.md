| №  | Термин                                 | Определение                                                                                     |
|----|----------------------------------------|-------------------------------------------------------------------------------------------------|
| 1  | Трейдер                                | Пользователь, который анализирует рынок и совершает торговые операции.                          |
| 2  | Аналитик                               | Пользователь, публикующий обзоры рынка и рекомендации для трейдеров.                            |
| 3  | Брокер                                 | Участник, исполняющий ордера клиента и подтверждающий сделки (через внешнюю торговую систему).  |
| 4  | Администратор системы                  | Пользователь, отвечающий за стабильность, доступы, мониторинг и безопасность платформы.         |
| 5  | Финансовый инструмент                  | Актив, доступный для наблюдения и торговли (акции, валюты и др.).                               |
| 6  | Тикер                                  | Короткий идентификатор финансового инструмента (например, AAPL).                                |
| 7  | Котировка                              | Актуальная цена/ценовые параметры инструмента, получаемые из рыночного фида.                    |
| 8  | Рыночный фид                           | Внешний источник рыночных данных, поставляющий котировки и обновления в реальном времени.       |
| 9  | Подписка на котировки                  | Механизм получения потока обновлений по выбранным инструментам.                                 |
| 10 | Ордер                                  | Поручение на покупку или продажу финансового инструмента.                                       |
| 11 | Рыночный ордер                         | Ордер, исполняемый по текущей доступной рыночной цене.                                          |
| 12 | Лимитный ордер                         | Ордер, исполняемый только по заданной цене или лучше.                                           |
| 13 | Статус ордера                          | Состояние ордера (создан, принят, частично исполнен, исполнен, отменен, отклонен).              |
| 14 | Исполнение ордера                      | Процесс выполнения ордера на торговой площадке/у брокера.                                       |
| 15 | Сделка                                 | Зафиксированный факт покупки/продажи инструмента в результате исполнения ордера.                |
| 16 | Подтверждение сделки                   | Сообщение о сделке от брокера/биржи (execution report / trade confirmation).                    |
| 17 | Портфель                               | Совокупность позиций и денежных средств пользователя.                                           |
| 18 | Позиция                                | Количество инструмента в портфеле с параметрами (средняя цена, текущая оценка и т.п.).          |
| 19 | Баланс                                 | Сумма денежных средств на счете пользователя в системе.                                         |
| 20 | Доступные средства                     | Часть баланса, доступная для новых ордеров с учетом резервов/комиссий.                          |
| 21 | История сделок                         | Журнал сделок пользователя за период, с фильтрами и детализацией.                               |
| 22 | PnL                                    | Прибыль/убыток по позиции или за период (Profit and Loss).                                      |
| 23 | Комиссия                               | Затраты на совершение операций (комиссия брокера/площадки/платежей).                            |
| 24 | Автоматическая стратегия               | Набор правил, по которым система может автоматически разместить ордер.                          |
| 25 | Триггер стратегии                      | Условие срабатывания стратегии (например, цена достигла порога).                                |
| 26 | Ценовой алерт                          | Уведомление о достижении инструментом заданного значения/условия.                               |
| 27 | Уведомление                            | Сообщение пользователю о событии (исполнение, отказ, алерт, срабатывание стратегии).            |
| 28 | Аналитический обзор                    | Публикация аналитика с оценкой рынка/инструмента.                                               |
| 29 | Рекомендация                           | Рекомендация аналитика (Buy/Sell/Hold) по инструменту или рынку.                                |
| 30 | Лента аналитики                        | Раздел/страница для просмотра обзоров и рекомендаций.                                           |
| 31 | Форум                                  | Раздел платформы для общения и обмена мнениями пользователей.                                   |
| 32 | Тема форума                            | Обсуждение, созданное пользователем в рамках форума.                                            |
| 33 | Сообщение                              | Публикация внутри темы форума.                                                                  |
| 34 | Роль пользователя                      | Набор прав доступа (трейдер/аналитик/брокер/админ).                                             |
| 35 | RBAC                                   | Управление доступом на основе ролей (Role-Based Access Control).                                |
| 36 | Аутентификация                         | Процесс подтверждения личности пользователя (логин/пароль, токены и т.п.).                      |
| 37 | MFA / 2FA                              | Многофакторная аутентификация (дополнительный фактор помимо пароля).                            |
| 38 | Шифрование данных                      | Защита данных при передаче и/или хранении с использованием криптографических методов.           |
| 39 | DDoS-защита                            | Механизмы защиты от распределенных атак отказа в обслуживании.                                  |
| 40 | Аудит / Журнал действий                | Логирование ключевых действий пользователей и системы для расследований и контроля.             |
| 41 | Интеграция с брокером                  | Взаимодействие платформы с брокерской/биржевой API для исполнения ордеров и получения статусов. |
| 42 | Интеграция с банком/платежным шлюзом   | Взаимодействие для пополнения/вывода средств (если предусмотрено сценарием).                    |
