# Контекст

Для проекта «Платформа для Трейдинга» требуется клиентское приложение (Web), которое обеспечивает:

- просмотр рыночных данных в реальном времени (котировки, стакан, лента сделок);
- выставление/отмена/изменение ордеров, просмотр статусов исполнения;
- управление портфелем, позициями и историей сделок;
- графики и базовая аналитика (индикаторы, фильтры, отчёты);
- уведомления (в т.ч. realtime) и безопасный вход (аутентификация/авторизация).

Нефункциональные требования: высокая отзывчивость UI, стабильная работа при большом количестве обновлений данных, поддержка современных браузеров, удобство масштабирования команды и расширения функциональности.

# Рассмотренные варианты

1. **React + TypeScript (SPA)**
2. **Vue.js**
3. **Angular**
4. **Svelte**
5. **Flutter Web** (единый стек с потенциальной мобильной версией)

# Решение

Выбрана технология **React + TypeScript** для клиентского Web-приложения.

# Обоснование

- **Большая и зрелая экосистема**: маршрутизация, управление состоянием, работа с WebSocket/стримингом, инструменты сборки и тестирования.
- **Компонентный подход** упрощает повторное использование UI-элементов (виджеты котировок, таблицы ордеров, карточки позиций, модальные формы).
- **Удобство для realtime-интерфейса**: React хорошо подходит для построения сложных динамических интерфейсов при правильной архитектуре состояния и мемоизации.
- **TypeScript снижает количество ошибок** в доменной логике (ордера/позиции/статусы) и повышает поддерживаемость кода.
- **Совместимость с выбранными практиками тестирования**: unit-тесты (Jest/RTL), E2E (Playwright) и CI-пайплайны.

# Недостатки

- Необходимо дисциплинированно проектировать состояние (иначе возможны «лишние» перерисовки).
- Потребуются соглашения по архитектуре фронтенда (структура модулей, стандарты компонентов, слои работы с API/событиями).

# Последствия

- Клиентское приложение реализуется на **React + TypeScript**.
- Рекомендуемый базовый стек (может уточняться по ходу проекта):
  - сборка: **Vite** (или Next.js при необходимости SSR/SEO);
  - маршрутизация: **React Router**;
  - состояние: **Redux Toolkit**;
  - запросы/кэш: **React Query**;
  - realtime: WebSocket-клиент + слой событий/буферизации;
  - UI-компоненты: MUI/AntD или собственный дизайн-кит;
  - графики: библиотека графиков Recharts.
- Вводятся правила качества: линтинг, форматирование, code review, обязательные тесты в CI.
